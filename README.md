# 🎯 Habit Tracker MVP (Telegram Bot + FastAPI + PostgreSQL)

Интерактивный кроссплатформенный чат-бот для автоматического трекинга и закрепления ежедневных привычек. Проект построен на базе современной MVP/MVC архитектуры, полностью контейнеризирован с помощью Docker и использует автоматическое управление миграциями.

---

## 🚀 Основная фича: Механика 21 дня (Автоперенос)
Вместо ресурсоемкого копирования строк в базе данных каждую полночь, в проект заложена элегантная бизнес-логика на уровне SQL-запросов (`SQLAlchemy`). 
* Привычка автоматически переносится на следующий день и отображается в списке целей пользователя до тех пор, пока отметка `✅ Выполнено` не наберет **ровно 21 раз**.
* Как только привычка успешно выполняется 21 раз, она считается закрепленной, переходит в архив и скрывается из ежедневного дашборда.

---

## 🛠 Технологический стек

* **Frontend (UI):** [pyTelegramBotAPI](https://github.com/eternnoir/pyTelegramBotAPI) (Python) — интерактивный интерфейс с динамическими `Inline`-меню и карточками задач.
* **Backend (API):** [FastAPI](https://fastapi.tiangolo.com/) — высокопроизводительный асинхронный веб-фреймворк для обработки CRUD-запросов.
* **Безопасность:** Авторизация на базе **JWT-токенов** (JSON Web Tokens) + безопасное хэширование данных через `passlib (sha256_crypt)`.
* **База данных:** PostgreSQL 15 (Alpine) + [SQLAlchemy 2.0](https://www.sqlalchemy.org/) (ORM-слой).
* **Миграции:** [Alembic](https://alembic.sqlalchemy.org/) — контроль версий и автоматическая генерация структуры таблиц.
* **Менеджер пакетов:** [Poetry](https://python-poetry.org/) — современное управление зависимостями проекта.
* **Окружение:** Docker + Docker Compose (изолированная сеть для трех сервисов).

## 📂 Архитектура проекта

Проект строго разделен на независимые слои (Каталоги соответствуют стандартам чистой архитектуры):

```text
habit_tracker/
├── backend/               # Серверная часть (FastAPI)
│   ├── api/               # Маршруты эндпоинтов (auth.py, habits.py)
│   ├── core/              # Безопасность и зависимости (security.py, dependencies.py)
│   ├── crud/              # Чистая бизнес-логика базы данных (user.py, habit.py)
│   ├── db/                # Конфигурация подключения и модели SQLAlchemy
│   ├── schemas/           # Валидация входных/выходных данных (Pydantic)
│   └── main.py            # Точка входа FastAPI сервера
├── bot/                   # Клиентская часть (Telegram-бот)
│   ├── main.py            # Обработчики команд, кнопок и NextStep-логика интерфейса
│   └── services.py        # Сетевой слой (клиент для отправки HTTP-запросов к FastAPI)
├── migrations/            # История и файлы миграций Alembic
├── Dockerfile.backend     # Сборка контейнера бэкенда
├── Dockerfile.bot         # Сборка контейнера бота
├── docker-compose.yml     # Оркестрация сервисов (Postgres, FastAPI, Bot)
├── alembic.ini            # Конфигурация Alembic
└── pyproject.toml         # Зависимости Poetry
